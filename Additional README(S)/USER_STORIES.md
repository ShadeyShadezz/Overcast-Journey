# OvercastJourney User Stories Pipeline

## Definition of Done
- Accepted by PO
- Tests pass
- Docs updated
- Deployed/staged

## Backlog (Prioritized)

### Must-Have (MVP)
**As an anonymous sender, I want drag/drop file upload so I can easily start transfers.**
Priority: Highest | Est: 3 | Status: Prototype

**As a sender, I want room code/QR generation so I can share transfer link.**
Priority: Highest | Est: 2 | Status: Implemented

**As a receiver, I want code/QR input so I can join transfers.**
Priority: Highest | Est: 2 | Status: Prototype

**As both, I want real-time progress indicators so I know transfer status.**
Priority: High | Est: 3 | Status: Basic impl

**As receiver, I want file preview/download so I control receipt.**
Priority: High | Est: 2 | Status: Implemented

**As user, I want encryption indicator so I trust security.**
Priority: High | Est: 1 | Status: Planned

### Nice-to-Have
**As sender/receiver, I want speed graphs so I see performance.**
Priority: Med | Est: 3

**As user, I want connection strength indicator so I troubleshoot.**
Priority: Med | Est: 2

**As user, I want transfer resume so interruptions don't lose work.**
Priority: Med | Est: 5

**As mobile user, I want gesture controls so natural UX.**
Priority: Low | Est: 3

**As user, I want transfer history so repeat access.**
Priority: Low | Est: 4

**As admin, I want transfer analytics so monitor abuse (anon hashes).**
Priority: Low | Est: 4

## Sprint Pipeline
```
Backlog → Ready → In Progress → Review → Done
```
